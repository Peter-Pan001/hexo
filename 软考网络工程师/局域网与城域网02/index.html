# 局域网互联

# 局域网互联设备

  • 两层网桥（生成树、源路由）、三层交换机、路由器。
  • 网桥要求 3 层以上的协议相同，1、2 层协议不同可互联

# 生成树网桥

  • 又叫透明网桥，IEEE802.1d, 生成树算法
  • 基本思想是在网桥之间传递 BPDU，比较参数，根据 STP 打开好端口，阻塞差的端口，沿着好的端口建立路径， 边走边拐弯，应用于以太网

生成树协议与算法参考文档：https://blog.csdn.net/weixin_43627022/article/details/105730616

# 生成树网桥步骤

  1. 确定根桥
  2. 确定根端口
  3. 确定指定桥
  4. 确定指定端口
  5. 阻塞剩余端口
  6. 形成无环网络

# 生成树网桥计算数据

  1. 确定根桥 ID
    • 优先级 + MAC 地址，都选最小
    • 优先级 0-65535，默认 32768（±4096）
  2. 确定根端口
    • 优先级 + 编号，都选最小的
    • 优先级 0-255，默认 128
  3. 三小原则
    • 优先级
    • MAC 地址
    • 通路费用

# 生成树端口的四种状态

  • Blocking（阻塞）
    • 接收 BPDU（网桥协议数据单元），不学习 MAC 地址，不转发数据帧。20s
  • Listening（侦听）
    • 接收 BPDU，不学习 MAC 地址，不转发数据帧，但交换机向其他交换机通告该端口，参与选举根端口或指定端口
  • Learning（学习）
    • 接收 BPDU，学习 MAC 地址，不转发数据帧
  • Forwarding（转发）
    • 正常转发数据帧
  • 总结：

阻塞 ————>20 秒 ————> 侦听 ————>15 秒 ————> 学习 ————>15 秒 ————> 转发

# 生成树种类和标准

种类 英文简称 标准
多生成树 MSTP IEEE802.1s
快速生成树 RSTP IEEE802.1w
生成树 STP IEEE802.1d
端口认证 基于用户 IEEE802.1x
  • 巧记： 多块生（人）口，是 s 我 w 弟 d 兄 x

# 源路由网桥

  • IEEE802.5，基本思想是发送探测帧到目的节点，返回路径以后沿着路径再传送，发送帧在这条路径上就传送，不在则发送广播，查询路径，选择最优路径再发送。 选好路再走

# 城域网简介

# 什么是城域网？

  • 城域网比局域网传输距离远，能覆盖整个城市。能够提供分组传输的数据、语音和视频等多媒体业务。更大的传输容量，更高的传输效率
  • 城域以太网
    • 以太网专用线
    • 以太网虚拟专线
    • 以太局域网服务
  • 无线城域网标准
    • WIMAX (802.16d 固定、802.16e 移动)
    • WIMAXII(802.16m 4G)

# Q-in-Q

  • 运营商网桥协议（PBP）IEEE802.1ad

# MAC-in-MAC

  • 运营商主干网桥（PBB）IEEE802.1ah

# 练习例题

  1. 生成树协议 STP 使用两个参数来选举根网桥。

    • 网桥优先级和 IP 地址
    • 链路速率和 IP 地址
    • 链路速率和 MAC 地址
    • 网桥优先级和 MAC 地址
    • 网桥优先级就是桥 ID

  2. 动态分配 VLAN 的方法不包括 。

    • 网络层协议
    • 网络层地址
    • 交换机端口
    • MAC 地址

  3. 在下面的标准中，定义快速生成树协议的是

    • 802.1d
    • 802.1w
    • 802.1s
    • 802.1x

  4. 在下面的标准中，支持端口认证的协议是

    • 802.1d
    • 802.1w
    • 802.1s
    • 802.1x

  5. 在生成树协议（STP）IEEE802.1d, 根据 来选中根交换机。

    • 最小的 MAC 地址
    • 最大的 MAC 地址
    • 最小的交换机 ID
    • 最大的交换机 ID

  6. 在当千兆以太网使用 UTP 作为传输介质时，限制单根的长度不超过 100m，其原因为

    • 信号衰减严重
    • 编码方式限制
    • 与百兆以太网兼容
    • 采用 CSMA/CD
    • 超过 100m 检测不到信号，而不是信号衰减

  7. IEEE802.1q 协议的作用是

    • 生成树协议
    • 以太网流量控制
    • 生成 VLAN 标记
    • 基于端口的认证
